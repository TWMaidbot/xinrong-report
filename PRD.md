## 1. 產品概述

### 1.1 背景
- 2026 年 1 月導入行動派工系統，使用不順利
- 現有系統操作繁瑣，與現場實際作業流程有落差
- 需要一套簡單、直覺的報工系統

### 1.2 目標

| 目標 | 衡量指標 |
|------|----------|
| 簡化報工流程 | 單次報工操作 < 30 秒 |
| 即時掌握進度 | 戰情看板即時更新 |
| 數據可追蹤 | 可匯出日/月報表（CSV） |
| 權限控管 | 依角色顯示對應功能 |

## 4. 功能模組

### 4.1 登入系統

**登入方式：**
- 輸入工號（如 P02、S01）
- 系統自動顯示對應姓名確認
- 登入後依權限顯示可用功能

**欄位：**
| 欄位 | 說明 |
|------|------|
| 工號 | 員工唯一識別碼 |
| 姓名 | 自動帶入（唯讀顯示） |
| 權限 | 自動帶入（唯讀顯示） |

---

### 4.2 報工功能（所有角色）

**報工流程：**
```
選擇機台 → 選擇工作類型 → 輸入數量 → 選擇協作者(選填) → 提交
```

**報工欄位：**
| 欄位 | 類型 | 說明 |
|------|------|------|
| 機台 | 下拉選單 / QR掃描 | 14 台機台選一 |
| 工作類型 | 下拉選單 | 依機台動態顯示 |
| 數量 | 數字 | +1/-1/+10/-10 快捷按鈕 |
| 單位 | 自動 | 依工作類型自動帶入 |
| 共同作業者 | 多選 | 點選姓名，可複選 |
| 備註 | 文字 | 選填 |
| 報工者 | 自動 | 登入者姓名 |
| 時間 | 自動 | 系統時間 |

**QR Code 掃描：**
- 每台機台貼有專屬 QR Code
- 掃描後自動帶入機台編號
- 格式：`https://[系統網址]/?m=M101`

---

### 4.3 戰情看板（主管、管理者）

**統計區塊：**
| 區塊 | 內容 |
|------|------|
| 今日報工次數 | 所有機台報工總次數 |
| 今日棧板產量 | 棧板類型總數 |
| 今日木箱產量 | 木箱類型總數 |

**機台進度：**
- 每台機台的今日報工次數
- 視覺化進度條

**時間軸：**
- 今日報工紀錄（時間倒序）
- 顯示：時間、機台、工作類型、數量、報工者

---

### 4.4 報工紀錄（主管、管理者）

**篩選條件：**
| 條件 | 類型 |
|------|------|
| 開始日期 | 日期選擇 |
| 結束日期 | 日期選擇 |
| 機台 | 下拉選單 |
| 員工 | 下拉選單 |

**報表匯出：**
| 報表 | 說明 |
|------|------|
| 日報表 | 指定日期的所有報工 |
| 月報表 | 指定月份的所有報工 |
| 篩選結果 | 依篩選條件匯出 |

**匯出格式：** CSV（UTF-8 BOM，Excel 相容）

**匯出欄位：**
時間、機台、機台名稱、工作類型、數量、單位、報工者、共同作業者、備註

---

### 4.5 使用者管理（僅管理者）

**功能：**
| 操作 | 說明 |
|------|------|
| 新增使用者 | 輸入工號、姓名、權限 |
| 編輯使用者 | 修改姓名、權限 |
| 刪除使用者 | 移除使用者（不可刪除自己） |

**權限選項：**
- 一般員工
- 主管
- 管理者

## 5. 系統架構

### 5.1 技術選型（Prototype 階段）

| 層級 | 技術 | 說明 |
|------|------|------|
| 前端 | 純 HTML/CSS/JS | 單一 HTML 檔案 |
| 資料儲存 | LocalStorage | 瀏覽器本地儲存 |
| 部署 | 靜態託管 | Catbox / Vercel / GitHub Pages |

### 5.2 正式版技術建議

| 層級 | 技術 | 說明 |
|------|------|------|
| 前端 | Vue.js / React | PWA 支援 |
| 後端 | Node.js / Python | RESTful API |
| 資料庫 | PostgreSQL / SQLite | 資料持久化 |
| 部署 | Vercel / Railway / 自架 | 可選雲端或本地 |
